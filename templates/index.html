How does it work?

Introduction

This nifty tool can be used to procure documentation and insights for a collection of Python scripts in your local directory. At the moment, the tool reads code comments with certain keywords and then renders them in a documentation format.

How to use

First off, place all your scripts in the project's scripts folder. Afterwards, depending on how you write your script, the documentation gets rendered as follows:

Stuff in your script How it will get rendered
Adding a multi-line comment (using ''') at the very top of your script Gets rendered as a Description in the Documentation
Adding a single-line comment with the word tags and then a list of tags separated by commas as a single-line comment Gets rendered as tags in the documentation. However, it will render only the first instance of where the tags comment is being used and will ignore the rest
Adding a single-line comment with the word context and then a list of tags separated by commas as a single-line comment Gets rendered as Context in the documentation. Use this to include useful clues and steps at any step of your code as these will all be listed in the order they appear in your code.
